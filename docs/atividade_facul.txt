Instruções
Envios de trabalho: grupo de defesa 05/06

Envios de trabalho: grupo de detecção de vulnerabilidade  12/06

Ambos grupos devem enviar/apresentar slides.

Grupo de defesa deve fazer um sistema de login com algum tipo de autenticação (o que usuário sabe, o que possui, o que é) e algum tipo de criptografia (cesar, vintagere, hash, hash com salto e etc).

Grupo de detecção deve analisar o código disponibilizado pelo grupo de defesa e  identificar vulnerabilidades ou a ausência das mesmas. Além disso, deve mencionar se essa vulnerabilidade está sujeito a algum tipo de ataque e qual/quais seria(m). Por exemplo: Analisando o código do grupo não foi encontrado restrições para tentativa de senhas (foto), o que torna o sistema vulnerável ao ataque de força bruta. O indicado seria ser feito essa alteração no código (foto).
